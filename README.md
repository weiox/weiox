## What I Care About

- Building tools that make real workflows easier, not just more automated
- Keeping systems understandable as they grow more capable
- Turning messy local setups into clean, inspectable, reproducible ones
- Writing technical explanations that help people think clearly, not just memorize terms

## Working Style

I like software that is:

- practical enough to survive real use
- simple enough to inspect and repair
- opinionated enough to reduce confusion
- documented well enough that future-me does not regret past-me
